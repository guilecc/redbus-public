Audio Bridge Setup

Enable system audio capture for meeting intelligence. Setup depends on your operating system.

Windows

Windows allows native system audio capture via WASAPI Loopback.

  • No additional drivers required.
  • Automatic "System Audio" capture.
  • Conflict-free simultaneous listening.

macOS

macOS requires a virtual audio driver (RedBusAudio) to intercept system sound.

  • Pre-installed custom HAL driver.
  • Automatically creates a Multi-Output Device on record.
  • Restores default output seamlessly on stop.

Troubleshooting

macOS "App is damaged" message

If you see a message saying RedBus is damaged, it's actually Gatekeeper blocking the unsigned app. Run this in your Terminal to fix:

sudo xattr -cr /Applications/RedBus.app

macOS Driver Installation

Follow these steps to manually install the audio bridge:

  1. Unzip the downloaded RedBusAudio-Driver.zip.
  2. Move the RedBusAudio2ch.driver folder to: /Library/Audio/Plug-Ins/HAL/
  3. Open your Terminal.
  4. Restart the audio system with: sudo launchctl kickstart -kp system/com.apple.audio.coreaudiod

Admin password will be required for the terminal command.
